2. Превращение большого массива в массив чанков. Без копирования, просто второй вариант массива с указателями по чанкам в основном.

Для создания кокмбинированного массива компонентов надо на другом треде создать новый комбинированный массив и вернуть его для системы.
Оптимизация - надо удалять комбинированный массив после использования системой, однако надо этот массив распокавать обратно в основные компоненты.

Пайплайн создания кобминированного массива:
Перед использованием кобминированного массива компонентов системой, надо сформировать такой массив(на другом потоке) путем копирования данных из нужных компонентов и расположения их в определенном порядке в комбинированном массиве. Массив должен быть в виде чанков. После использования массива системой, надо отчистить из памяти этот массив.

Примечание. Создавать массив из ссылок на элементы нужных компонентов бессмысленно. Надо формировать новый массив, что бы элементы в памяти распологались последовательно. В случае ссылок, элементы разбросаны по памяти, что противоречит cache friendly архитектуре.
std move and std swap тоже бессмысленны.

система запрашивает компоненты. если нужен компонент комбинированный, то создается комбинированный массив. После использования системой, комбинированный массив распаковывается обратно в изначальные компоненты.

Другой вариант делать изначально комбинированые компоненты, но тогда теряется гибкость, т.к. некоторым системам может понадобиться только один тип элемента из компонента.
